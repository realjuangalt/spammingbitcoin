How to gate content

You are a Site. After enroll you get a secret portal link (bookmark it — no password). Agents unlock your URLs with SHA-256d; you accrue sats and we pay your Lightning address via Blink when unpaid ≥ {{ min_sats }} sats.

Fees: the Pool keeps {{ pool_fee_percent }}% of gross hashrate earnings; your Site receives {{ site_payout_percent }}%. This is disclosed before you enroll — not a standard ~1–2% mining-pool take.

1 · Enroll via API (no UI)

curl -sS {{ api_base }}/v1/publishers \
  -H 'content-type: application/json' \
  -d '{"origin":"https://your-site.example","display_name":"Your Site","access_zero_bits":18,"lightning_address":"you@blink.sv"}'

Response includes portalUrl (save it) and apiKey for your server.

2 · Flow

#WhoAction
1AgentGET your gated URL
2SiteCreate job → respond 402 + PAYMENT-REQUIRED
3AgentMine → receiptToken
4AgentRetry with PAYMENT-SIGNATURE
5SiteSettle with Pool → serve content

Bind every job to the same absolute resource URL for create / mine / settle.

3 · Create a job

curl -sS {{ api_base }}/v1/jobs \
  -H 'content-type: application/json' \
  -d '{"publisher_id":"pub_…","resource":"https://your-site.example/premium","access_zero_bits":18}'

4 · Return 402

Header PAYMENT-REQUIRED = base64url(JSON). Put job fields in accepts[0].extra, set facilitator to {{ api_base }}, and point minerHint at:

{{ api_base }}/static/mine_share.py

python3 /tmp/mine_share.py --api {{ api_base }} \
  --job-id JOB --prefix76 HEX --target 0x… \
  --publisher pub_… --resource https://your-site.example/premium \
  --receipt-out -

5 · Settle / unlock

curl -sS {{ api_base }}/v1/settle \
  -H "authorization: Bearer sk_…" \
  -H 'content-type: application/json' \
  -d '{"receipt_token":"…","publisher_id":"pub_…","resource":"https://your-site.example/premium"}'

On 200, serve the gated content. Reference: demo vault.

6 · Balance & payouts

curl -sS {{ api_base }}/v1/publishers/me -H "authorization: Bearer sk_…"

Or open your secret /s/… portal. Unpaid sats pay out to your Lightning address (Blink) at ≥ {{ min_sats }} sats.

7 · Security

  • Treat the portal URL like a password — don’t publish it.
  • Keep apiKey on the server only (settle / jobs).
  • Same resource string for job, mine, and settle.
  • Prefer Pool settle so receipts are single-use.
